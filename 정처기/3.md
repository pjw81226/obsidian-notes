정형기술검토 (FTR)
- 정형 기술 검토(FTR, Formal Technical Review)는 소프트웨어 개발 과정에서 만들어진 코드나 설계서 같은 결과물의 품질을 높이기 위해 수행하는 **공식적인 검토 회의**
	
- **인스펙션 (Inspection):** 가장 공식적인 검토 방법입니다. **중재자(Moderator)** 를 두고 정해진 체크리스트에 따라 아주 엄격하게 오류를 찾습니다.
    
- **워크스루 (Walkthrough):** 개발자가 주도하여 동료들에게 내용을 설명하며 오류를 찾는 비교적 비공식적인 회의입니다. 지식 공유의 성격이 강합니다.

- **검증 (Verification):** "제품을 **올바르게** 만들고 있는가?"를 따집니다. 개발자 관점에서 명세서대로 잘 구현되었는지 확인하는 과정입니다.
    
- **확인 (Validation):** "올바른 **제품**을 만들었는가?"를 따집니다. 사용자 관점에서 사용자가 원래 원했던 기능을 수행하는지 확인하는 과정입니다.

- 7대 지침
	1. **제품 검토의 집중:** 사람이 아닌 **제품(코드, 문서)** 의 잘못된 점을 비판해야 합니다. (작성자 공격 X)
	2. **의제 제한:** 미리 정해진 안건만 다룹니다. 삼천포로 빠지는 것을 막기 위함입니다.
	3. **논쟁과 반박 제한:** 끝없는 토론보다는 에러를 찾아내는 데 집중합니다.
	4. **문제 영역 명확화:** 무엇이 문제인지 확실히 짚고 넘어갑니다.
	5. **참가자 수 제한:** 보통 3~5명 정도의 소규모 인원으로 밀도 있게 진행합니다.
	6. **시간 제한:** 집중력이 떨어지지 않게 보통 2시간 이내로 끝냅니다.
	7. **사전 준비 필수:** 검토자들은 미리 내용을 보고 와야 합니다.

정규화
- 1정규형
	- 모든 도메인이 **원자 값** 으로만 구성
	- 한 칸에 여러 개의 값이 들어있는 다중 값을 별도의 행으로 분리하거나 테이블을 나눈다.
- 2정규형
	- 부분 함수적 종속성을 제거
	- 기본키가 복합키일 때 기본키의 일부에만 종속되는 속성을 분리한다.
	- (A,B) -> C 이면서 A -> C 인 관계를 제거한다.
- 3정규형
	- 이행적 함수적 종속성을 제거
	- A->B 이고 B->C 일때 A -> C 가 성립하는 관계를 제거
	- C는 A가 아닌 B에의해 결정 됨 
	- 기본키가 아닌 속성 간에 종속성이 존재하는 경우를 분리한다.
- BCNF
	- 모든 결정자가 후보키여야한다. (결정자이면서 후보키가 아닌 함수 종속 제거)
	- A -> B 일 때 A가 후보키가 아니면 제거
- 4정규형
	- 다치 종속 제거
	- 독립적인 두 종류 이상의 다중 값이 있다면 이로인한 중복 제거
	- 예를들어 A가 수강중인 과목과 A가 들어가있는 동아리가 컬럼에 각각 있다면 둘다 기본키에 대해 여러개의 값을 가질 수 있으므로 이런 부분을 제거하는 것 (두개의 테이블로 분리)
- 5정규형
	- 조인 중복 제거
	- 오직 **후보키**를 통해서만 조인 종속이 성립하게 만든다.

분산 데이터베이스 시스템
- 분산 데이터베이스의 목표 : 4대 투명성
	1. 위치 투명성 : 저장되어있는 물리적 위치를 몰라도 접근할 수 있어야한다.
	2. 중복 투명성 : 데이터가 여러곳에 중복 저장되어있어도 하나의 데이터처럼 사용해야한다.
	3. 병행 투명성 : 여러 사용자가 동시에 트랜잭션을 실행해도 결과에 이상이 없어야한다.
	4. 장애 투명성 : 특정 부분에 장애가 발생해도 전체 시스템은 동작해야한다.
- 구성요소
	-  분산 처리기 : 각 사이트에서 자체적인 처리 능력을 가지는 CPU/컴퓨터
	- 분산 데이터베이스 : 각 사이트에 분산되어 저장된 실제 데이터들의 집합
	- 통신 네트워크 : 분산 처리기들을 연결하여 데이터를 주고받는 통신

IPv4 IPv6
- 옥텟 : 8비트의 묶음
- IPv4
	- 32비트, 8비트 씩 네묶음. 
	- 10진수로 표현
	- IPv4 환경에서는 **DHCP(Dynamic Host Configuration Protocol)** 서버가 사용자가 수동으로 IP 주소를 입력하지 않아도, 네트워크에 접속하면 시스템이 알아서 주소를 할당해준다.
	- 헤더의 크기는 가변적
	- 유니캐스트는 1:1 통신방식
	- 유니캐스트, 멀티캐스트 브로드캐스트 모두 지원
- IPv6
	- 128비트, 16비트씩 8부분
	- 16진수로 표현 각 부분은 콜론으로 구분 (2001:0db8:85a3:...)
	- **헤더는 40 옥텟의 고정 길이**
	- 주소 자동 설정: Plug & Play 기능처럼 사용자의 단말을 네트워크에 손쉽게 접속시킬 수 있는 자동 설정 기능을 지원
	- 보안성이 강화됨

OSI 7계층 프로토콜
- **7. Applicaiton Layer 응용 계층 : 사용자가 네트워크에 접속할 수 있게 해주는 인터페이스 역할**
	- HTTP 웹 문서 전송
	- FTP (File Transper) : 파일 전송
	- SMTP / POP3, IMAP : 메일 발송, 수신
	- DNS : 도메인 이름을 IP 주소로 변환
	- SSH / Telnet : 원격 접속

 - **6. Presentation Layer 표현 계층 : 데이터의 형식 결정, 암호화, 압축을 담당** 
	 - JPEG, MPEG, GIF: 이미지/영상 형식
	 - ASCII, 유니코드: 문자 인코딩 
	- SSL/TLS: 암호화 (응용 계층과 혼용되기도 함)

- **5. Session Layer 세션 계층 : 응용 프로세스 간의 통신 연결(세션)을 설정, 유지, 종료**
	- RPC (Remote Procedure Call) : 원격 프로시저 호출
	- NetBIOS : 네트워크 기본 입출력 시스템

- 4. **Transport Layer 전송 계층 : End to End 신뢰성 있고 정확한 데이터 전송 담당**
	- TCP
	- UDP (비 연결형)

- **3. Network Layer 네트워크 계층 : 데이터를 목적지까지 가장 빠른 경로(라우팅)로 전달**
	- IP (IPv4, IPv6): 주소 지정 및 경로 선택
	- ICMP: 오류 보고 및 네트워크 상태 진단 (Ping)
	- ARP: IP 주소를 물리적 MAC 주소로 변환

- **2. Data Link Layer 데이터링크 계층 : 인접한 노드 간의 물리적 전송과 오류 제어를 담당**
	- HDLC, PPP : 광대역 전송(WAN)용 프로토콜
	- LLC, MAC : 이더넷 관련 하위 계층
	- 이더넷 (Ethernet): LAN 표준

- **1. Physical Layer 물리 계층 : 전기적, 기계적 신호를 전송**
	- RS-232, DSL: 물리적 인터페이스 표준


DoS 공격
- Ping of Death
	- 규격보다 훨씬 큰 ICMP 패킷을 전송
	- 네트워크를 통해 전송될 때 큰 패킷은 Fragment로 쪼개지는데 수신 측에서 이 수많은 조각을 다시 조립하는 과정에서 과부하가 걸리거나 버퍼 오버플로우가 발생하여 시스템이 다운
- Smurf Attack
	- 출발지 IP를 공격 대상의 IP로 속인 후(Spoofing), 네트워크의 브로드캐스트(Broadcast) 주소로 ICMP Echo Request를 보낸다.
	- 브로드캐스트를 받은 네트워크 안의 수많은 컴퓨터가 응답 패킷(Echo Reply)을 한꺼번에 공격 대상에게 보냄으로써 대역폭을 마비시킨다.
	- ==**그니까 내 패킷의 출발지를 공격대상으로 해놓고 브로드캐스트 패킷 보내서 공격 대상이 응답을 한번에 많이 받게하는 방법이다.**==
- SYN Flooding
	- 3-Way Handshake의 약점을 이용한다.
	- 수많은 SYN 패킷을 보내고 공격지점으로부터 SYN + ACK 을 받는다. 공격자는 이에 대한 ACK을 보내지 않음으로써 계속 대기 상태로 만들어 서버의 접속 가능 자원을 꽉 채워버린다.
- Land Attack 
	- 패킷을 보낼 때 출발지 IP 주소와 목적지 IP 주소를 동일하게 설정하여 전송
	- 패킷을 받은 시스템은 응답을 보낼 때 자기 자신에게 다시 보내게 되고, 이 과정이 무한 루프에 빠지면서 시스템 리소스를 급격히 소모하게 만든다.

Secure 코딩에서 입력 데이터의 보안 약점과 관련한 설명
- 사용자로부터 입력받는 데이터를 제대로 검증하지 않았을 때 발생하는 보안 사고의 종류

- XSS 크로스사이트 스크립트
	- 게시판이나 URL 매개변수에 악성 스크립트(JavaScript)를 삽입하여 사용자의 브라우저에서 실행되게 하는 공격
	- 검증되지 않은 외부 입력 값에의해 브라우저에서 악의적인 코드가 실행되게 함
- 운영체제 명령어 삽입
	- OS Command Injection
	- 운영체제 명령어 파라미터 입력값이 적절한 사전검증을 거치지 않고 사용되어 공격자가 운영체제 명령어를 조작
- 자원 삽입
	- 외부의 입력값이 시스템 자원(파일, 포트, 프로세스 등)의 식별자로 사용될 때, 이를 적절히 검증하지 않아 공격자가 자원의 위치나 종류를 마음대로 조작할 수 있는 보안 약점

윈도우 파일시스템
- 과거에 사용하던 **FAT(File Allocation Table)** 
- 현재 Windows의 표준인 **NTFS(New Technology File System)

- NTFS 
	- **보안성 향상** : NTFS는 파일 및 폴더별로 접근 권한을 설정할 수 있어 보안이 매우 강력
	- **대용량 지원** : FAT32가 한 파일당 최대 4GB까지만 지원하는 것과 달리, NTFS는 테라바이트(TB) 단위의 대용량 볼륨과 파일을 효율적으로 관리
	- **안정성 및 복구** : 시스템 오류 발생 시 파일 시스템을 자가 복구하는 기능을 갖추고 있으며, 데이터 압축 및 암호화 기능을 자체적으로 지원
	- **오버헤드 발생** : 보안이나 복구 기능이 많다 보니 시스템 리소스를 더 사용한다. 그래서 아주 저용량 볼륨에서는 오히려 속도가 저하되거나 효율이 떨어질 수 있다.


DPI
- 보통의 방화벽이나 네트워크 장비는 패킷의 **헤더(Header)** 부분(출발지/목적지 IP, 포트 번호 등)만 보고 통과 여부를 결정
- DPI는 헤더뿐만 아니라 패킷의 **데이터 부분(Payload/Contents)** 까지 전부 파악. 사용자가 어떤 내용을 보내는, 악성 코드가 포함되어 있는지 '내용물'을 직접 확인하는 기술
