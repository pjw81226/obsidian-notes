#### 여러 모형
- 폭포수 모형
	- 보헴이 제시, 고전적
	- 이전단계로 못돌아감
	- **요구사항 반영이 어려움**
- 나선형 모형
	- 나선을따라 돌듯이 점진적으로 소프트웨어 개발
	- 계획수립 → 위험분석 → 개발, 검증 → 고객평가 **반복적 수행**


- 에자일 모형 핵심 가치
	- 계획보다 **변화에 대한 대응**을 중요하게 생각함
	- 프로세스와 도구보다 **개인과 상호작용**에 더 가치를 둠
	- 방대한 문서보다 **실행되는 SW**에 더 가치를 둠
	- 계약 협상보다는 **고객과 협업**에 더 가치를 둠
		
	- 계획 → 개발 → 출시 → 피드백 (반복적)

- 에자일 모형 주요 방법론
	- **스크럼(Scrum), XP, FDD, 칸반(Kanban), Lean**
- XP (eXtreme Programming)
	- **의사소통, 단순성, 용기, 피드백, 존중**
		- 두사람이 컴퓨터 한대로 프로그래밍
		- TDD(Test Driven Development) : 테스트 코드를 먼저 작성하고 기능개발
		- 코드를 자주 통합하고 빌드/테스트 자동화
		- 리펙토링
		- 짧은 주기로 배포, 고객피드백

>구조적 다이어그램 : 시스템의 정적인 뼈대를 표현
- 구조적 다이어그램의 종류 
	- **클래스** 다이어그램 : **클래스, 속성, 메소드**를 포현
	- **객체** 다이어그램 : 특정 시점의 **객체 관계**를 스냅샷처럼 표현
	- **컴포넌트** 다이어그램 : 코드 라이브러리, 파일, 모듈 등 **물리적인 구성 요소**들의 구조와 의존 관계를 표현
	- **배치** 다이어그램 : 결과물이 물리적인 하드웨어에 어떻게 배치되어 실행되는지 표현
	- **복합체 구조** 다이어그램 : 클래스나 컴포넌트의 **내부 구조**와 외부와의 상호작용 지점(Port)을  표현
	- **패키지** 다이어그램 : 서로 관련된 요소들을 패키지로 그룹화해 계층구조와 의존성을 표현

> 행위 다이어그램 : 시스템의 동적인 움직임을 표현
- 행위 다이어그램의 종류
	- **유스케이스** 다이어그램 : 사용자 관점, 시스템이 제공하는 기능(요구사항)을 표현
	- **순차** 다이어그램 : 객체간에 주고받는 메세지를 시간의 흐름에따라 표현
	- **커뮤니케이션** 다이어그램 : 순차 다이어그램과 유사, 객체간의 연결관계를 더 강조
	- **상태** 다이어그램 : 객체 생명주기동안 발생하는 상태변화와 원인을 표현
	- **활동** 다이어그램 : 시스템의 로직, 업무 프로세스의 **처리흐름**을 순서대로 표현
	- **상호작용 개요** 다이어그램 : 활동다이어그램과 비슷, 흐름의 각 단계가 **다른 상호작용 다이어그램** 표현
	- **타이밍** 다이어그램 : 객체 상태 변화와 시간 제약 사항을 **타임라인** 형태로 정밀하게 표현

#### 사용자 인터페이스
- 사용자 인터페이스의 특징
	- 사용자의 **편리성과 가독성**을 높여준다.
	- **작업시간단축**
	- 업무에 대한 **이해도를 높임**
	- **사용자 중심**으로 설계되어 있다.

- 사용자 인터페이스 구분
	- CLI : 명령과 출력이 텍스트 형태로 이루어지는 인터페이스
	- GUI : 아이콘이나 메뉴를 마우스로 선택해 작업을 수행하는 그래픽 환경 인터페이스
	- NUI : 사용자의 말이나 행동으로 기기를 조작하는 인터페이스

- 사용자 인터페이스의 기본 원칙
	- **직관성** : 누구나 쉽게 이해하고 사용할 수 있어야함.
	- **유효성** : 사용자의 목적을 정확하고 완벽하게 달성해야 함
	- **학습성** : 누구나 쉽게 배우고 익힐 수 있어야한다.

#### 설계
- 목업
	- 와이어프레임보다 좀 더 실제 화면과 유사하게 만든 정적인 형태의 모형
	- 시각적으로 구성요소만 배치, **실제로 구현되지 않음**

- ISO / IEC 9126 의 품질 특성 (품질 표준 지침)
	- **기능성** : 요구사항을 정확하게 만족하는 기능을 제공하는지 여부를 나타냄
	- **신뢰성** : 요구된 기능을 오류없이 수행할 수 있는 정도를 나타냄
	- **사용성** : 사용자가 쉽게 배우고 사용할 수 있는 정도를 나타냄
	- **이식성** : 다른환경에서도 얼마나 쉽게 적용할 수 있는지 정도를 나타냄
