
## 네트워크 장비 (계층별)

**L2 스위치** 
- MAC 주소를 기반으로 전송. 데이터 링크 계층에서 동작하며, 가장 기본적인 스위치

**L3 스위치**
- IP 주소를 기반으로 라우팅 기능을 수행합니다. L2 기능에 라우터 기능이 합쳐진 형태

**L4 스위치**
- 포트 번호를 기반으로 트래픽을 분산합니다. 로드 밸런싱 기능이 핵심, TCP/UDP 프로토콜을 이해한다.

**L7 스위치**
- 애플리케이션 계층(HTTP URL, 쿠키, 이메일 내용 등)까지 분석하여 정교한 부하 분산과 보안 필터링을 수


## 보안 공격 기법

**DoS**
- Ping of Death 
	- 허용 범위 이상의 **큰 패킷** 을 전송해서 네트워크 마비
- Smurf Attack
	- **ICMP Echo** 패킷과 I**P 브로드캐스트**를 이용해 엄청난 양의 데이터를 한 곳에 집중시킴
	- 에코를 개많이 보내거나 출발지를 공격대상으로 바꿔놓고 브로드캐스트를 보내서 응답 많이 받도록
- SYN Flooding
	- **3-Way Handshaking** 과정을 악용하여 서버의 접속 자원(Backlog Queue)을 점유


**네트워크 공격**
- 스니핑
	- 네트워크상에 흘러 다니는 패킷을 **몰래 엿듣는** 행위
- 스푸핑
	- 자신의 식별 정보를 **속여서(위장)** 신뢰받는 시스템인 척함
- Session Hijacking
	- 이미 연결된 두 시스템 사이의 **세션 정보를 가로채** 접속 권한을 뺏음


**애플리케이션 및 시스템 보안 공격**
- SQL Injection
	- 데이터베이스 쿼리문에 악성 코드를 삽입하여 정보를 유출하거나 DB를 조작
- XSS
	- 게시판 등에 악성 스크립트를 삽입하여, 이를 열람한 사용자가 의도하지 않은 동작을 하도록함
- Buffer Overflow
	- 메모리의 할당된 범위를 넘치게 하여 프로그램의 실행 흐름을 바꾸거나 관리자 권한을 획득



## 소프트웨어 개발 및 품질 관리

##### SOA (Service Oriented Architecture, 서비스 지향 아키텍처)
대규모 컴퓨터 시스템을 구축할 때, 업무의 일련의 과정을 서비스라는 단위로 정의. 서비스를 조립해서 전체 시스템을 만드는 아키텍처

- SOA의 주요 계층 구성
	- **표현층 (Presentation Layer)**
		- 사용자와 시스템이 만나는 지점.
		- 웹페이지나 앱의 화면 등 사용자가 직접 보고 조작하느 부분을 담당
	- **프로세스층 (Process Layer)**
		- 여러개의 서비스 조합해서 하나의 workflow를 만든다.
		- 비즈니스 로직을 구현
	- **비즈니스 층 (Business Layer)**
		- 핵심 업무 로직, 데이터 처리 계층
		- 실제 비즈니스 가치를 만들어내는 서비스들이 정의
	- **서비스층 (Service Layer)**
		- 독립적으로 실행 가능한 최소 단위의 기능을 제공한다.

##### CBD (Component Based Development, 컴포넌트 기반 개발) 
컴포넌트를 가져와서 조합해 소프트웨어를 만드는 방법론. 이미 만들어진 부품을 재사용

- 특징
	- **생산성 향상** : 이미 검증된 부품을 가져다 쓰므로 개발기간 단축
	- **재사용성 (Reusability)** : 한번 만든 컴포넌트는 다른 프로젝트에서 재사용 가능
	- **유지보수 용이** : 특정 기능 문제발생하면 해당 컴포넌트만 교체
	- **독립성** : 각 컴포넌트 캡슐화, 인터페이스로만 소통


##### CMM (Capability Maturity Model, 소프트웨어 프로세스 성숙도 모델)
소프트웨어 개발 조직이 얼마나 체계적으로 일하는지 그 성숙도를 측정하는 기준
1. **1단계 : 초기단계**
	- 표준화된 프로세스가 전혀 없고 오로지 개인의 능력에 의존
2. **2단계 : 관리단계**
	- 최소한의 프로젝트 단위 관리가 이루어진다. 비용 일정 기능 등을 추적하기 시작
3. **3단계 : 정의 단계**
	- 프로젝트 단위를 넘어, 조직 전체의 표준 프로세스가 문서화 되어있다.
4. **4단계 : 정량적 관리 단계**
	- 프로세스의 품질을 숫자로 측정하고 관리
5. **5단계 : 최적화 단계**
	- 데이터를 바탕으로 지속적인 개선에 집중

##### SPICE (ISO / IEC 15504)
`Software Process Imporvement and Capability dEtermination`
소프트웨어 프로세스를 개선하고 능력을 측정하기 위한 국제 표준
CMM은 대학에서 만든 모델, SPICE는 이를 바탕으로 전세계에서 통용될 수 있도록 만든 **국제적인 기준**

1. **Level 0 불완전** : 프로세스가 구현되지 않았거나 목적을 달성하지 못한 상태
2. **Level 1 수행** : 프로세스가 수행되고 목적이 달성된 상태
3. **Level 2 관리** : 프로세스가 규칙에 따라 수행되고 산출물이 관리되는 상태
4. **Level 3 확립** : 조직의 표준 프로세스가 정의되어 사용되는 상태
5. **Level 4 예측** : 프로세스가 일관되게 수행되며 정량적으로 관리되는 상태
6. **Level 5 최적화** : 지속적인 개선을 통해 비즈니스가 목적을 만족시키는 상태



## 인프라 및 저장장치
##### DAS NAS SAN

- **DAS (Direct Attached Storage)**
	- ==연결 방식== : 전용 광케이플로 1:1 직접 연결, 망이 없음
	- ==접근단위== : 블록 단위
	- ==파일시스템== : 서버(host)가 관리
	- ==장점== : 설치 간편, 전용 대역폭 사용
	- ==단점== : 확장성 부족, 공유 불가

- **NAS (Network Attached Storage)**
	- ==연결 방식== : LAN
	- ==접근단위== : 파일 단위
	- ==파일시스템== : 스토리지(NAS)가 관리
	- ==장점== : 파일 공유 가능, 구축 비용 저렴
	- ==단점== : 네트워크 병목현상 발생 가능

- **SAN (Storage Area Network)**
	- ==연결 방식== : 전용 광케이블 (FC 망)
	- ==접근단위== : 블록 단위
	- ==파일시스템== : 서버(host)가 관리
	- ==장점== : 고속 전용 망, 확장성 / 성능 최고
	- ==단점== : 구축 비용 매우 비쌈 관리 복잡

**망의 차이**
- DAS 
	- 네트워크 없음
	- 서버 뒤에 하드디스크 바로 꽂는 식.
- NAS
	- 회사 내 공유 폴더
	- 기존 사무실 랜선을 그대로 이용
- SAN
	- 스토리지만을 위한 별도의 전용 광네트워크를 따로 사용
	- 서버들이 이 네트워크를 타고 스토리지를 공유한다.


##### SDDC (Software Defined Data Center)
모든 인프라 자원을 소프트웨어로 가상화하여 관리하는 지능형 데이터 센터

- 핵심 구성요소
	- **SDC (Software Defined Computing)** : CPU, 메모리 같은 계산 자원을 가상화
	- **SDN (Software Defined Networking)** : 네트워크 제어부와 전송부를 분리하여 소프트웨어로 관리
	- **SDS (Software Defined Storage)** : 물리적 스토리지를 가상화하여 하나의 커다란 자원풀로 관리

- 주요 특징
	- **가상화 기반** : 모든 자원을 소프트웨어로 정의하고 가상화
	- **자동화** : 인력의 개입 없이 소프트웨어 조작만으로 자원을 할당하고 관리
	- **유연성과 확장성** : 하드웨어 제약없이 필요할 때 즉시 자원을 늘리거나 줄일 수 있음
	- **하드웨어 독립성** : 특정 제조사 하드웨어에 종속되지 않는 표준화된 범용서버 사용

##### MAC DAC RBAC
누가, 무엇에, 어떤 권한을 가질지 결정하는 방법론

**DAC** (Discretionary Access Control, 선택적 접근 통제)
- **핵심** : 데이터의 소유자가 권한을 마음대로 부여
- **특징** : 사용자의 신분에 따라 권한을 준다. "내가 만든 파일이니까 A한테만 읽기 권한 줄게"
- **장점** : 구현이 용이하고 유연하다.
- **단점** : 소유자가 권한 부여를 남발하면 보안이 취약해질 수 있다.

**MAC** (Mandatory Access Control, 강제적 접근 통제)
- **핵심** : 사용자가 아니라 시스템이 미리 정해놓은 보안등급에 따라 통제
- **특징** : 주체의 보안 등ㅇ급과 객체의 보안등급을 비교해 접근을 허용. 비밀등급이 낮으면 아무리 소유자라도 1급 비밀을 볼 수 없다.
- **장점** : 매우 강력한 보안을 유지하며 중앙 집중적
- **단점** : 설정이 복잡하고 유연성이 떨어짐

**RBAC** (Role Based Access Control, 역할 기반 접근 통제)
- **핵심** : 사용자의 신분이 아니라 조직내에서 맡은 역할에 따라 권한을 준다.
- **특징** : 중앙 관리자가 권한을 부여. 사용자가 퇴사하거나 부서가 바뀌어도 역할만 수정하면 되므로 관리가 편함. 현대의 거의 모든 웹서비스나 기업시스템에서 사용

**소프트웨어 비용 산정 및 관리**
- 기능점수 산정의 5대 요소
	- 외부 입력 (EI) : 데이터 입력
	- 외부 출력 (EO) : 보고서 출력
	- 외부 조회 (EQ) : 명령어 수
	- 내부 논리 파일 (ILF) : 내부 데이터 파일
	- 외부 인터페이스 팡리 (EIF) : 외부 시스템과의 데이터 연동

**임계 경로**
- 처음부터 끝까지 가장 오래걸리는 경로

**RIP 특징**
- **거리 벡터 라우팅 프로토콜**
- **벨만 포드 알고리즘** : 인접한 라우터끼리 주기적으로 라우팅 정보 교환, 최단경로를 갱신할 때 사용
- **최대 홉 수 제한** : 최대 15홉 까지 허용
- **거리단위** : 목적지까지 거쳐가는 라우터의 개수인 홉카운트를 기준으로 경로를 설정

- **소규모 네트워크용** : 오버헤드가 적지만 최대 홉수 제한 때문에 대규모 네트워크에는 제한
