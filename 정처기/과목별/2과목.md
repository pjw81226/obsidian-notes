## 1. 인터페이스 보안 및 테스트 도구
##### 인터페이스 보안
- SMTP 는 메일 보낼 때 사용하는 Applicaiton 계층 프로토콜
- IPsec, SSL, S-HTTP 모두 보안 프로토콜

##### 인터페이스 구현 검증 도구
-  xUnit, STAF NTAF 는 소프트웨어의 기능을 테스트 하고 검증하는데 사용하는 테스트 프레임워크/도구
- ESB(Enterprise Service Bus)는 서로 다른 애플리케이션 간의 통합을 도와주는 미들웨어 아키텍처



## 2. 애플리케이션 테스트 기법
##### 인수테스트
- 알파 검사 : 개발자의 장소에서 사용자가 개발자 앞에서 행하는 테스트.
- 베타 검사 : 개발자 없이 사용자의 실제 환경에서 이루어지는 ㅔㅌ스트

##### 테스트 오라클
- 테스트 결과가 맞는지 틀린지 판단하기 위해 미리 정의된 참값을 넣어 비교하는 기법
	- 토탈 오라클 : **모든** 입력값에 대해 기대하는 결과를 확인한다. 
	- 샘플링 오라클 : 특정 몇몇 입력값에 대해서만 결과를 확인한다
	- 휴리스틱 오라클 : 특정 입력값은 정확하게 확인, 나머지는 통계적 추정으로 처리
	- 일관성 검사 오라클 : 애플리케이션 변경 전후의 실행 결과가 일관성 있는지 확인



## 3. 빌드 자동화 및 형상 관리
##### 빌드 자동화 도구

- 비용 관리는 코드의 형상을 관리하는 SCM의 직접적인 역할이 아님



## 4. McCabe 복잡도 계산

## 5. 소스코드 품질 분석 도구

**정적 분석 도구** (정형 분석 도구)
- 소프트웨어를 실행하지 않고 소스코드 자체를 분석하는 도구
- 특징 : 코딩 표준 준수 여부, 잠재적 결함, 보안 취약점, 복잡도

- 도구
	- pmd checkstyle, findbugs, cppcheck, SonarQube


**동적 분석 도구** (비정형 분석 도구)
- 소프트웨어를 실행하면서 메모리 누수, 스레드 결함 등을 분석
- 특징 : 실제 실행 환경에서의 성능 측정, 메모리 사용량, 실행 경로 확인 등을 수행한다.

- 도구
	- Valgrind, Avalanche
