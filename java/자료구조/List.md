### ArrayList
- 구조 : 크기가 가변적인 리스트
- 장점 : 인덱스를 통한 데이터 조회(`get`)가 매우 빠르다.
- 단점 : 중간에 데이터를 추가하거나 삭제할 때 뒤의 데이터들을 밀어내야 하므로 속도가 느리다.
- 용도: 데이터의 변경이 적고 조회가 빈번한 경우

### LinkedList
- 구조: 데이터가 노드 형태로 연결되어 있다.
- 장점: 데이터의 추가/삭제가 매우 빠르다. (주소 값만 바꾸면 됨)
- 단점: 특정 위치의 데이터를 찾으려면 처음부터 순차적으로 탐색해야 하므로 조회가 느리다.
- 용도: 데이터의 추가/삭제가 빈번한 경우.
